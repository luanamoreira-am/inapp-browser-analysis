# inapp-browser-analysis
Análise de um teste A/B/C relacionado ao In-App Browser, o navegador interno usado no app para abrir lojas parceiras.
Este documento apresenta a análise detalhada de um teste A/B/C realizado no In-App Browser (IAB) do aplicativo Méliuz. O objetivo principal é interpretar o problema de produto, reconstruir o teste a partir dos dados brutos fornecidos, calcular métricas de sucesso com precisão, e formular uma recomendação fundamentada para a próxima melhoria. Além disso, será documentado o processo de trabalho com IA, demonstrando a abordagem analítica e crítica adotada.

# Agente de Análise de Testes A/B — In-App Browser

**Organização do Agente no Desafio Méliuz**

Este documento detalha a arquitetura e o processo de trabalho do agente autônomo durante o desafio Méliuz, abordando a [interpretação das instruções](system_prompt.md), os [mecanismos de validação](validation.md), os critérios de qualidade aplicados, as [estratégias de reaproveitamento](reusable_template.md) e o processo de revisão dos resultados.

## 1. [Interpretação e Aplicação das Instruções](system_prompt.md)

As instruções iniciais foram cruciais para moldar a abordagem do agente. A compreensão do papel como Product Manager de uma plataforma de cashback e benefícios, com foco no produto Shopping (cashback de lojas) no app, direcionou a análise para iniciativas com base em impacto para o negócio, valor para o usuário e alinhamento estratégico. A cultura do Méliuz, com seu Propósito Transformador Massivo e o pilar "Priorize e Simplifique", foi internalizada para guiar as decisões e a comunicação.

Foi dada ênfase à necessidade de ser analítico e aprofundar-se no problema antes de propor soluções, garantindo uma progressão lógica das ideias. A restrição de evitar jargões de marketing e manter um tom natural/conversacional também foi um guia constante na elaboração das respostas e documentos.
